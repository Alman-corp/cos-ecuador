### A04 · Implementar RLS completo (UPDATE/DELETE) en PostgreSQL
**¿Qué es?** Row Level Security (RLS) es una funcionalidad de PostgreSQL que impide que un usuario acceda a filas que no le pertenecen. El COS es multi-tenant: cada fila contiene un `tenant_id` y las políticas RLS garantizan que un cliente jamás vea datos de otro. Actualmente solo existen políticas `SELECT` e `INSERT`; faltan las de `UPDATE` y `DELETE`, lo que constituye una vulnerabilidad crítica donde un tenant podría modificar o borrar datos de otro.
**Estado actual:** Políticas parciales. Las auditorías lo marcan como **CRÍTICO** (hallazgo S1).
**Esfuerzo:** 🟡 Medio

### A05 · Configurar Elasticsearch para indexación documental
**¿Qué es?** Elasticsearch es un motor de búsqueda distribuido que permite buscar texto libre a velocidad de milisegundos dentro de miles de documentos (contratos, balances, reportes). Sin él, las búsquedas dependen de `LIKE` en PostgreSQL, lo cual es lento y no admite búsqueda por relevancia ni sinónimos. El `docker-compose.yml` ya lo incluye pero no hay código que cree índices ni que indexe documentos.
**Estado actual:** Contenedor definido en [docker-compose.yml](file:///c:/Users/hp/OneDrive/Desktop/TESIS/tesis%20carlos/CONSULTORIA/cos/docker-compose.yml) (línea 64). Sin mappings ni pipelines de indexación.

### A06 · Desplegar Qdrant para almacenamiento vectorial (RAG)
**¿Qué es?** Qdrant es una base de datos vectorial que almacena *embeddings* — representaciones numéricas del significado de textos. Cuando un usuario pregunta al agente IA "¿cuál es el riesgo de liquidez de este cliente?", el sistema busca en Qdrant los fragmentos de documentos más similares semánticamente a esa pregunta (no por palabras clave, sino por *significado*). Es la columna vertebral del RAG (Retrieval-Augmented Generation).
**Estado actual:** Diseñado en blueprints. Sin contenedor en docker-compose ni código de integración.

## CATEGORÍA 3 · MOTORES DE CÁLCULO (Python / FastAPI)

### A17 · Motor DCF real (Discounted Cash Flow)
**¿Qué es?** El DCF es el método estándar para valorar una empresa. Proyecta los flujos de caja libres futuros (5-10 años) y los descuenta al presente usando el WACC (Weighted Average Cost of Capital). Calcula: FCFF (Free Cash Flow to Firm), Valor Terminal (modelo de Gordon Growth o Exit Multiple), Enterprise Value y Equity Value. Es el corazón de la consultoría de M&A.
**Estado actual:** El frontend [valuation/page.tsx](file:///c:/Users/hp/OneDrive/Desktop/TESIS/tesis%20carlos/CONSULTORIA/cos/apps/web/src/app/consultor/dashboard/valuation/page.tsx) consume datos de una API que devuelve `dcfResult` — pero la API calcula valores estáticos. Se necesita un motor en Python con QuantLib/NumPy que reciba estados financieros reales como input.
**Esfuerzo:** 🔴 Alto

---

### A18 · Motor Monte Carlo real
**¿Qué es?** La simulación de Monte Carlo ejecuta miles de escenarios (10,000-100,000) variando aleatoriamente parámetros clave (crecimiento de ventas, margen operativo, tasa de interés, WACC) según distribuciones estadísticas (Normal, Triangular, Lognormal). El resultado es una distribución de probabilidad del valor de la empresa, no un solo número. Permite decir "hay 85% de probabilidad de que el Equity Value esté entre $1.8M y $3.2M".
**Estado actual:** El archivo [tools/financial_tools.py](file:///c:/Users/hp/OneDrive/Desktop/TESIS/tesis%20carlos/CONSULTORIA/cos/services/ai-orchestrator/tools/financial_tools.py) (10 KB) contiene `run_montecarlo_cashflow` con una implementación funcional de Monte Carlo para cash flow. Necesita extenderse para valuación DCF completa y generación de heatmaps de sensibilidad.
**Esfuerzo:** 🟡 Medio

### A19 · Motor MIDAS de nowcasting macroeconómico
**¿Qué es?** MIDAS (Mixed Data Sampling) es un modelo econométrico que combina datos de frecuencias diferentes para predecir en tiempo real. Ejemplo: usa indicadores diarios (tasa de interés, precio del petróleo, tipo de cambio) para predecir el PIB trimestral *antes de que se publique oficialmente*. Para una consultora en Ecuador, permite responder "¿cómo va a cerrar el PIB este trimestre?" usando datos que ya están disponibles hoy.
**Estado actual:** Diseñado en [BLUEPRINT_PLAN_CREACION.md](file:///c:/Users/hp/OneDrive/Desktop/TESIS/tesis%20carlos/CONSULTORIA/BLUEPRINT_PLAN_CREACION.md) §3.2.2.2. Sin implementación.
**Esfuerzo:** 🔴 Alto

### A20 · Motor Bayesian VAR con Conditional Forecasting
**¿Qué es?** Un VAR Bayesiano es un modelo que captura las relaciones dinámicas entre múltiples variables macroeconómicas simultáneamente (PIB, inflación, tasa de interés, crédito). Lo "condicional" permite fijar la trayectoria de una variable (ej. "la Fed sube tasas 50bps") y proyectar el impacto en el resto. Usa PyMC para inferencia y genera distribuciones predictivas completas, no pronósticos puntuales.
**Estado actual:** Solo documentación en blueprints. Sin código.
**Esfuerzo:** 🔴 Alto

### A25 · Implementar RAG con trazabilidad ISD
